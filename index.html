Polymer 3 In Action

By now we know the why and how of Polymer 3. In short, using better tools and sticking to web standards. Let's take a closer look at the syntax of a Polymer 3 component. We'll be making a simple component from scratch, closely following the hands-on that Google provides.

From scratch


> yarn init
Given that you have globally installed Yarn using NPM, you can do a yarn init. This will give you a tool to setup your program, similar to the polymer-cli. Sadly, the generators of the polymer-cli are not yet updated with a polymer 3 version. That's why we'll do this by hand. Running this gives you a package.json

package.json


{
  "name": "polymer3-preview",
  "version": "1.0.0",
  "flat": true,
  "dependencies": {
	"@polymer/polymer": "next",
	"@webcomponents/webcomponentsjs": "^1.0.13"
  }
}
This, ofcourse, is the package.json created. There is not so much spectecular going on. Just mind flat, which is necessary. And imagine that all your dependencies, whether they used to be NPM or Bower, are now in this file.

index.html


<html>
  <head>
    <script src="node_modules/@webcomponents/webcomponentsjs/webcomponents-loader.js"></script>
    <script type="module" src="src/my-app.js"></script>
  </head>

  <body>
    <my-app></my-app>
  </body>
</html>
Next, lets create our application. The index.html, is the file that glues all together. We load the webcomponentsjs polyfill. We add the module for our app to be. And we'll add the tag for our app. Note that we don't mention polymer here.

my-app.js



import { Element as PolymerElement }
  from '../node_modules/@polymer/polymer/polymer-element.js';

export class MyApp extends PolymerElement {
  ...
}

customElements.define('my-app', MyApp);
Then, moving to the my-app.js, the ES Module. We import polymer from the node_modules, using the standard syntax for ES Modules. Then, as you know of Polymer 2, we create a class. This is exposed by the export syntax.

my-app.js (2)


export class MyApp extends PolymerElement {
	
  static get template() {
		return `
			<style>
			  ...
			</style>
      <h1>Hello [[name]]</h1>
      <img src="polymer3.jpg">`;
  }
  ...
What is interesting, is ofcourse the HTML and CSS inside the module. The template is now implemented as a get method. This is ofcourse a simple example. I am a bit worried about having big css and html code in your template. Not a lot is said about CSS, though. For example on the topic of global custom styles? I knew that Vaadin is doing a lot with templating sites with CSS. I asked them, but they don't have a clear answer on this topic. But this is it: The main change to our components.

my-app.js (3)


  constructor() {
    super();
    this.name = '3.0 preview';
  }

  static get properties() {
    name: {
      type: String
    }
  }
}
So for the rest of the component, e.g. the constructor and properties stay the same. Same counts for listeners, and the callback functions.
